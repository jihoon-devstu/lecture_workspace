①  로그인 페이지

1. ID 입력
2. PWD 입력
3. 로그인 / 회원가입
    - 아이디 혹은 패스워드가 일치하지 않는 경우 , 아이디 or 패스워드가 일치하지 않습니다 문구 출력.
    
    *** 문구→알림 출력
    
    - 로그인 시 , 계정 정보에 따라 어드민 계정인 경우엔 관리자 페이지 출력 및 이동 / 나머지 계정(일반유저)의 경우엔 쇼핑몰 메인 페이지로 이동.
    - 어드민 계정은 처음부터 존재. 회원가입으로 추가되는 계정은 전부 일반 유저 계정.
    -> 따라서 User 테이블은 1개 (관리자, 일반회원)

---

② 회원가입 페이지

- 필드
    1. ID 입력 (중복확인 버튼)
    2. Password
    3. Confirm Password
    4. Name
    5. Email
    6. Address
- 기능
    1.  ID 중복확인 체크 (Unique 제한)
    2. 비밀번호 암호화
    3. 비밀번호 1번 더 입력 및 확인(비밀번호가 일치합니다 , 비밀번호가 일치하지 않습니다 출력)
    
      4.  회원가입 클릭 시, 메인 페이지를 출력
    

---

③ 쇼핑몰 메인 페이지

1. 유틸 바 - 로그인 정보 및 마이페이지
2. 네비 바 - 상품 카테고리 (전체, 의류, 뷰티 , 식품 , 디지털)
3. 상품 검색기능 추가할 건지 안할건지 확인 필요
4. 상품 작은 썸네일 이미지 , 상품명 , 금액 순서로 출력
    
    -> 메인 화면은 상품 테이블 위에서부터 순서대로 출력
    -> 유틸바에서 마이페이지 클릭 시 마이페이지로 화면전환
    -> 네비바의 카테고리 및 상품 썸네일 각 이미지마다, 호버 효과 추가
    -> 상품 옵션도 추가 (패션의 경우 색상 , 사이즈 옵션 추가)
    -> 메인 페이지에서 상품은 상품 이미지, 상품명, 가격 출력
    


---

④ 상품 상세 페이지

-> 상품 왼쪽에 크게 1장

->  상품명(Text) ,상품 금액(Text -> 옵션(특히 수량)에 따라 금액 변경) , 상품 옵션(Option)_수량,색상,사이즈
->  상품 상세 설명 출력
->  주문하기 버튼
->  기본적으로 내가 등록한 주소가 출력되게 하되 , 변경할 수 있도록 수정기능 추가

*** 주문서 페이지가 따로 없기 때문에 주소 수정을 원할 시 마이페이지에서 수정

⑤마이 페이지

1. 주문내역
-> (주문확인중 , 상품준비중, 배송중 , 배송완료 조회 및 주문 취소)
2. 회원 수정 (이름, 주소 , 비밀번호 수정)
3. 회원 탈퇴

---

⑥ 쇼핑몰 관리 페이지

1. 상품 목록 조회 _ 메인 페이지
-> 상품 카테고리 필터 , 스크롤을 내리며 상품별 재고수량 확인 , 등록된 상품 수정 , 삭제 , 품절처리 버튼
-> 상품 수정버튼을 누른 경우 , 상품 신규등록 화면에 해당 상품의 정보들이 다 채워져서 출력 (수정 후 변경 완료 버튼)
-> 품절버튼을 누른 경우 , 쇼핑몰 메인 페이지에서 Sold out 으로 해당 상품을 클릭하지 못하도록 변경.

2. 상품 신규 등록
-> 상품 카테고리 , 상품명 , 가격 , 상품 썸네일 2장 (상세페이지용 큰거1개, 메인 노출용 작은거1개) , 상품 상세 내용 (제조일 , 유통기한 , 재고 수량 , 상품 설명) 등록
-> 관리자만 보이도록 상품 재고도 추가로 등록 (고객들이 확인 가능한 페이지엔 노출되지 않도록) - 그래야 관리자가 재고 몇개 남았는지 확인 후 , 신청하기 편할 것 같아서

3. 고객 주문 관리
-> 어떤 고객(고객이름,전화번호,주소) 이 어떤 상품을 어떤 옵션(수량,색상,사이즈 등) 으로 주문했는지 확인 후
처리 현황 수정 - 주문확인 , 상품 준비중 , 배송중 , 배송완료
